# Behavioral Features

This document defines the behavioral variables generated by the NAVIDSON simulation.

These features represent the observable activity of employees across Gladney Pharmaceuticals and serve as inputs for machine learning models.

Features are grouped by behavioral domain rather than department. Individual employees will exhibit different baseline values depending on their role, historical behavior, and current organizational context.

# Feature Engineering Notes

Not all behavioral features are directly observed.

NAVIDSON distinguishes between three categories of features:

## Raw Features

Collected directly from enterprise systems.

Examples include:

- Login Time
- VPN Login
- Files Accessed
- USB Connected
- Teams Messages
- Device Health Score

## Contextual Features

Describe the environment in which behavior occurs.

Examples include:

- Active Organizational Event
- Business Travel
- PTO
- FDA Submission Window
- Software Rollout Active
- Performance Review Period

## Derived Features

Calculated from historical behavior and observed activity.

Examples include:

- Behavioral Drift
- Baseline Deviation
- Historical Similarity
- Context Alignment Score
- Confidence Accumulation Score
- Anomaly Score

# NAVIDSON Philosophy

Behavioral features are designed to help explain employee activity rather than simply identify anomalies.

Observed behavior is evaluated against an employee's historical baseline while incorporating organizational context.

The objective is to generate explainable recommendations for human analysts—not automated accusations.

Behavioral changes should always be interpreted in the context of the employee's role, current organizational events, and accumulated historical evidence.
