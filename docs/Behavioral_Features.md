# Behavioral Features

This document defines the behavioral features generated by PROJECT NAVIDSON.

These features describe observable employee behavior, organizational context, and derived behavioral metrics generated throughout the simulation.

Rather than representing isolated events, each feature contributes to a broader behavioral profile that can be interpreted over time. Together, these features provide the foundation for behavioral analytics, anomaly detection, and explainable cybersecurity research.

Behavioral features are organized by domain rather than department. Individual employees exhibit different baseline patterns based on their role, historical activity, and current organizational context.

# Feature Engineering Notes

Not all behavioral features are directly observed.

NAVIDSON distinguishes between three categories of features:

## Raw Features

Collected directly from enterprise systems.

Examples include:

- Login Time
- VPN Login
- Files Accessed
- USB Connected
- Teams Messages
- Device Health Score

## Contextual Features

Describe the environment in which behavior occurs.

Examples include:

- Active Organizational Event
- Business Travel
- PTO
- FDA Submission Window
- Software Rollout Active
- Performance Review Period

## Derived Features

Calculated from historical behavior and observed activity.

Examples include:

- Behavioral Drift
- Baseline Deviation
- Historical Similarity
- Context Alignment Score
- Confidence Accumulation Score
- Anomaly Score

# NAVIDSON Philosophy

Behavioral features are intended to describe employee activity, and not to determine intent.

Individual observations rarely provide sufficient evidence for investigation. Instead, NAVIDSON evaluates behavior within the context of historical baselines, organizational events, departmental expectations, and accumulated behavioral evidence.

The objective is not to identify malicious employees, but to provide explainable behavioral assessments that support informed human decision-making.

Behavior should always be interpreted within the organizational context in which it occurs.
